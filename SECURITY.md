# 管理后台安全配置说明

## 🔐 修改管理员密码

### 当前默认密码
```
bst2024
```

### 如何修改密码

1. 打开文件：`src/views/Login.vue`
2. 找到第 36 行左右：
```javascript
const ADMIN_PASSWORD = 'bst2024'; // 请修改为你自己的密码
```
3. 将 `'bst2024'` 改为你想要的密码，例如：
```javascript
const ADMIN_PASSWORD = 'mySecurePassword123';
```
4. 保存文件，前端会自动重新编译

### 安全建议

⚠️ **重要提示**：
- 密码至少8位，包含字母、数字和特殊字符
- 不要使用容易被猜到的密码（如：123456、admin、password等）
- 定期更换密码
- 不要将密码提交到公开的代码仓库

## 🔒 登录机制

### Token 有效期
- 登录后 Token 有效期：**24 小时**
- 超过 24 小时需要重新登录

### 存储位置
- Token 存储在浏览器的 localStorage 中
- 清除浏览器数据会导致需要重新登录

### 访问控制
以下页面需要登录才能访问：
- `/admin` - 管理后台主页
- `/admin/new` - 写新博客
- `/admin/edit/:id` - 编辑博客

未登录用户访问这些页面会自动跳转到登录页面。

## 🕵️ 隐藏式访问（安全增强）

**管理后台已从导航栏完全移除**，访客看不到任何管理入口。

### 管理员访问方式

只有你知道这些地址：

1. **登录页面**
   ```
   https://你的域名.com/login
   或 http://localhost:3001/login（开发环境）
   ```

2. **管理后台**
   ```
   https://你的域名.com/admin
   或 http://localhost:3001/admin（开发环境）
   ```

3. **写新博客**
   ```
   https://你的域名.com/admin/new
   或 http://localhost:3001/admin/new（开发环境）
   ```

### 推荐做法

- 📌 将管理地址加入浏览器书签
- 🔐 不要在公开场合分享管理地址
- 🚫 访客完全看不到管理入口，增强安全性

## 📝 未来改进建议

如果需要更高的安全性，可以考虑：
1. 使用后端 API 进行密码验证（而不是前端验证）
2. 使用 JWT（JSON Web Token）
3. 添加验证码功能防止暴力破解
4. 记录登录日志
5. 支持多个管理员账号
6. 修改管理路径为更隐蔽的地址（如：`/secret-admin-panel-xyz`）
